# Econ/Finance Exam Prep

Generates retrieval-practice material (quizzes, flashcards, problem sets) and runs the student through it like an actual test, rather than just explaining the material upfront — that's a different skill's job.

## 1. Get the essentials, then start

Ask in one go for whatever isn't already clear from context: topic/chapter, format (multiple choice / short answer / calculation problems / flashcards / mixed), difficulty, and roughly how many items. Don't interrogate beyond that — if the user says "quiz me on price elasticity, 10 questions," that's enough to start.

## 2. Generate the set

Build a mix appropriate to the topic and course level:
- **Conceptual** items test understanding of definitions and mechanisms ("why does a price ceiling cause a shortage").
- **Calculation** items require working a number (elasticity, NPV, a ratio, a regression coefficient interpretation).
- **Graph-it** items describe a scenario and ask what happens to a curve/equilibrium — good for micro/macro theory courses.

Vary difficulty across the set rather than making every item the same. Don't reveal the answer key alongside the questions — that defeats the point of practice.

## 3. Run it like a real test

- For calculation and short-answer items, present one at a time and wait for an attempt before grading — that's the retrieval practice that actually builds recall. Batch presentation is fine for multiple choice or flashcards where the format is naturally rapid-fire.
- Resist the pull to start explaining the concept before they've attempted the question. If the student is using this skill, they've already decided they want practice, not a first explanation — if it turns out they need the concept taught from scratch, say so and suggest a more explanation-oriented approach instead of quietly switching modes mid-quiz.

## 4. Grade with explanation, not just right/wrong

For each answer: say whether it's correct, then explain *why* — and for a wrong answer, name the specific misconception rather than just restating the correct answer (e.g., "you shifted the curve instead of moving along it — a price change is a movement along the demand curve, not a shift"). This is where the actual learning happens in a quiz format.

## 5. Track weak spots and re-serve them

Within the session, note which items or sub-topics the student missed, and circle back to a variant of those later in the same session (not the identical question — a structurally similar one) rather than only covering each topic once. If they want a longer study session, offer to keep going on the topics they're shakiest on.

## 6. Flashcard format specifically

Present as Term → (wait for the student's recall attempt) → Definition/explanation. Ask them to self-rate (knew it / shaky / didn't know) and weight which cards resurface sooner based on that, the way spaced-repetition systems do — don't just cycle through the full deck in a fixed order every time.
